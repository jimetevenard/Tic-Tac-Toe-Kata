Tic-Tac-Toe Kata

The Goal of this Kata is to implement a working Tic-Tac-Toe game.

It can be a nice training on coding event-driven programs, and more globally on how to decompose a complex problem into simpler ones.

Rules

    The rules of the Tic-Tac-Toe game are the following:

  • a game is over when all cells are taken
  • a game is over when all cells in a column, a row or a diagonal are taken by a player
  • a player can take a cell if not already taken
  • players take turns taking cells until the game is over
  • there are two player in the game (X and O)

In the editor below, you can implement the game in JavaScript.

For simplicity, the HTML + CSS integration of the board is already done.
Your goal is to focus on algorithmic logic.

To make it easier,
we suggest you to decompose the work into two main step

Step 1 - Just a board for human players

The game will have to allow two challengers to play together (on the same computer) in the respect the rules listed above.

The game must prevent users from making illegal actions, like playing an already taken cell, and must detect game-ending situations (a player has won, or all cells are taken)

Step 2 - Play against the machine

The second step would be to programmatically implement a computer player that one can confront.

It does not necessarily have to be a good player, just following the rules can be fine
